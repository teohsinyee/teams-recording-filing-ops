# Scope

This change set defines the minimum rules for weekly approval payloads, copy fallback, and copied-file detection.

## Required Behavior

- `outputs/recording-approval-ui/app.js` must include `sourceDir` and `destinations` in the saved approval payload.
- `work/recording-approval-ui/copy-approved-recordings.ps1` must accept approval files that either include `destinations` or rely on config fallback.
- The copy script must treat a missing `sourcePath` as non-existent instead of throwing.
- `work/recording-approval-ui/generate-recordings-json.ps1` must check for an exact renamed MP4 in the same target folder when an exact VTT match is already known.
- Only after that exact check fails may the generator fall back to date-and-duration matching.

## Validation Rules

- A reviewed approval file generated by the current UI must be accepted by the copy flow.
- An older approval file without `destinations` must still be accepted by the copy flow if config destinations exist.
- A recording with both renamed MP4 and renamed VTT already in target must be marked as already copied.

## Out Of Scope

- Rewriting the target-suggestion rules
- Broad duplicate-detection redesign
- Transcript-generation behavior inside Stream
